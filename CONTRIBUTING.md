# Contributing

Thanks for helping with this project.

## Rules

- Keep changes focused and small.
- Use clear code and readable names.
- Run the tool before opening a PR.
- Run the tests with `python -m unittest discover tests` to ensure everything passes.
- Be respectful in issues and discussions.

## Good PRs

- Fix one thing per PR.
- Describe what changed and why.
- Mention any behavior changes.

## Style

- Match the existing code style.
- Keep the tool simple unless a change really needs more complexity.
